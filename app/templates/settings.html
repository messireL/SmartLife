Режим работы с Tuya Cloud

Для домашнего сценария теперь есть нормальный ручной режим: SmartLife не ест лимит Tuya в фоне, а `local_key` и `IP` подтягиваются только по кнопке на конкретном устройстве. Экономичный и стандартный режимы оставлены для тех случаев, когда облако всё же нужно опрашивать автоматически.
Рекомендация для trial и LAN-first сценария: выбрать Ручной. Тогда при появлении облака ты сам решаешь, по каким устройствам сделать точечный fetch key/IP, а фоновый sync лимит не подъедает.

Tuya-сцены и home_id

В ручном режиме SmartLife не делает автоматический cloud refresh и не тратит лимит Tuya на фоновый опрос. Когда облако оживёт, нужный key/IP подтягивай только вручную по устройству на вкладке Локально.
Укажи один или несколько home_id через запятую или с новой строки. Именно по этим домам SmartLife будет подтягивать Tap-to-Run сцены и Tuya-автоматизации.

Где хранятся облачные настройки

Параметры подключения к облаку теперь хранятся в PostgreSQL (`app_settings`), а не в `.env`. Команда `configure-tuya` записывает их в базу. Legacy secrets остаются только как одноразовый источник для первого импорта при обновлении старого стенда.

Изменить настройки облака

Настройки сохраняются в PostgreSQL (`app_settings`). Secret в форме обратно не показывается; пустое поле оставит текущий secret без изменений.
